# <a name="setting-up-the-director-in-lync-server-2013"></a>Configuration du directeur dans Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

_**Dernière modification de la rubrique :** 2014-05-05_

Si vous activez l’accès pour les utilisateurs externes en déployant des serveurs Edge, vous pouvez déployer un directeur. Un directeur est un serveur exécutant Microsoft Lync Server 2013 qui authentifie les demandes des utilisateurs, mais n’héberge aucun compte d’utilisateur. À présent, cela n’est pas obligatoire, mais il est très utile si vous craignez les performances et que vous souhaitez aider à rationaliser les demandes d’authentification. Si vous décidez qu’il s’agit d’une bonne idée pour votre organisation, les étapes de configuration d’un directeur ou d’un pool directeur sont similaires à la configuration d’un pool frontal Enterprise Edition ou d’un serveur Standard Edition. Une fois que vous avez défini vos Directors dans le générateur de topologie, vous devez effectuer les étapes décrites dans cette section.
