<span data-ttu-id="b5560-109">L’authentification consiste à donner des informations d’identification des utilisateurs à un serveur approuvé.</span><span class="sxs-lookup"><span data-stu-id="b5560-109">Authentication is the provision of user credentials to a trusted server.</span></span> <span data-ttu-id="b5560-110">Skype Entreprise Server utilise les protocoles d’authentification suivants, en fonction de l’état et de l’emplacement de l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="b5560-110">Skype for Business Server uses the following authentication protocols, depending on the status and location of the user.</span></span>
  
- <span data-ttu-id="b5560-p104">Le **protocole MIT Kerberos, version 5** pour les utilisateurs internes disposant d’informations d’identification Active Directory. - <span data-ttu-id="b5560-p105">Le **protocole NTLM** pour les utilisateurs disposant d’informations d’identification Active Directory qui se connectent à partir d’un système d’extrémité situé à l’extérieur du pare-feu de l’entreprise.     > [!NOTE]
    > <span data-ttu-id="b5560-116">Le protocole NTLM offre une protection contre les attaques plus faible que Kerberos, c’est pourquoi certaines organisations limitent son utilisation.</span><span class="sxs-lookup"><span data-stu-id="b5560-116">NTLM protocol offers weaker attack protection than Kerberos, so some organizations minimize usage of NTLM.</span></span> <span data-ttu-id="b5560-117">Par conséquent, l’accès à Skype Entreprise Server peut être limité aux clients internes ou connectés via une connexion VPN ou DirectAccess.</span><span class="sxs-lookup"><span data-stu-id="b5560-117">As a result, access to Skype for Business Server might be restricted to internal or clients connected through a VPN or DirectAccess connection.</span></span> 
  
- <span data-ttu-id="b5560-p107">Le **protocole Digest** pour les utilisateurs dits « anonymes ». Digest authentication is not used for other client interactions.</span></span>
    
<span data-ttu-id="b5560-121">L’authentification Skype Entreprise Server se compose de deux phases :</span><span class="sxs-lookup"><span data-stu-id="b5560-121">Skype for Business Server authentication consists of two phases:</span></span>
  
1. <span data-ttu-id="b5560-122">Une association de sécurité est établie entre le client et le serveur.</span><span class="sxs-lookup"><span data-stu-id="b5560-122">A security association is established between the client and the server.</span></span>

<span data-ttu-id="b5560-p109">L’approbation d’un utilisateur est attachée à chaque message envoyé par l’utilisateur, et non à l’identité de l’utilisateur proprement dite. <span data-ttu-id="b5560-128">Vous pouvez utiliser d’autres contraintes, si nécessaire, pour empêcher les utilisateurs disposant d’informations d’identification valides délivrées par un partenaire fédéré d’accéder à l’ensemble des privilèges accordés aux utilisateurs internes.</span><span class="sxs-lookup"><span data-stu-id="b5560-128">Users with valid credentials issued by a federated partner are trusted but optionally prevented by additional constraints from enjoying the full range of privileges accorded to internal users.</span></span>
  
<span data-ttu-id="b5560-129">Les protocoles ICE et TURN utilisent également l’authentification Digest, comme décrit dans le document RFC TURN de l’IETF.</span><span class="sxs-lookup"><span data-stu-id="b5560-129">The ICE and TURN protocols also use the Digest challenge as described in the IETF TURN RFC.</span></span>
