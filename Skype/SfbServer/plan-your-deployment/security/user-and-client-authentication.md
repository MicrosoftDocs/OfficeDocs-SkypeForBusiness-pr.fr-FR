<span data-ttu-id="080c3-106">Un confiance est un utilisateur dont les références ont été authentifiées par un serveur de confiance dans Skype pour Business Server 2015.</span><span class="sxs-lookup"><span data-stu-id="080c3-106">A trusted user is one whose credentials have been authenticated by a trusted server in Skype for Business Server 2015.</span></span> <span data-ttu-id="080c3-107">Ce serveur est généralement un serveur Standard Edition, Enterprise Edition serveur frontal ou directeur.</span><span class="sxs-lookup"><span data-stu-id="080c3-107">This server is usually a Standard Edition server, Enterprise Edition Front End Server, or Director.</span></span> <span data-ttu-id="080c3-108">Skype pour Business Server s’appuie sur les Services de domaine Active Directory comme référentiel principal unique et fiable des informations d’identification de l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="080c3-108">Skype for Business Server relies on Active Directory Domain Services as the single, trusted back-end repository of user credentials.</span></span>
  
<span data-ttu-id="080c3-109">L’authentification consiste à fournir des informations d’identification d’utilisateur à un serveur approuvé.</span><span class="sxs-lookup"><span data-stu-id="080c3-109">Authentication is the provision of user credentials to a trusted server.</span></span> <span data-ttu-id="080c3-110">Skype pour Business Server utilise les protocoles d’authentification suivants, en fonction de l’état et l’emplacement de l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="080c3-110">Skype for Business Server uses the following authentication protocols, depending on the status and location of the user.</span></span>
  
- <span data-ttu-id="080c3-111">**Protocole de MIT Kerberos version 5 de sécurité** pour les utilisateurs internes avec des informations d’identification Active Directory.</span><span class="sxs-lookup"><span data-stu-id="080c3-111">**MIT Kerberos version 5 security protocol** for internal users with Active Directory credentials.</span></span> <span data-ttu-id="080c3-112">Kerberos nécessite une connectivité client Active Directory Domain Services, c’est pourquoi il ne peut pas être utilisé pour authentifier les clients à l’extérieur du pare-feu d’entreprise.</span><span class="sxs-lookup"><span data-stu-id="080c3-112">Kerberos requires client connectivity to Active Directory Domain Services, which is why it cannot be used for authenticating clients outside the corporate firewall.</span></span>
    
- <span data-ttu-id="080c3-113">**Protocole NTLM** pour les utilisateurs avec des informations d’identification Active Directory qui sont connectent à partir d’un point de terminaison à l’extérieur du pare-feu d’entreprise.</span><span class="sxs-lookup"><span data-stu-id="080c3-113">**NTLM protocol** for users with Active Directory credentials who are connecting from an endpoint outside the corporate firewall.</span></span> <span data-ttu-id="080c3-114">Le service Edge d’accès transmet les demandes d’ouverture de session à un directeur, le cas échéant, ou à un serveur frontal pour l’authentification.</span><span class="sxs-lookup"><span data-stu-id="080c3-114">The Access Edge service passes logon requests to a Director, if present, or a Front End Server for authentication.</span></span> <span data-ttu-id="080c3-115">Le service Edge d’accès lui-même n’effectue aucune authentification.</span><span class="sxs-lookup"><span data-stu-id="080c3-115">The Access Edge service itself performs no authentication.</span></span>
    
    > [!NOTE]
    > <span data-ttu-id="080c3-116">Le protocole NTLM offrant une protection plus faible que Kerberos contre les attaques, certaines organisations minimisent l’utilisation de NTLM.</span><span class="sxs-lookup"><span data-stu-id="080c3-116">NTLM protocol offers weaker attack protection than Kerberos, so some organizations minimize usage of NTLM.</span></span> <span data-ttu-id="080c3-117">Par conséquent, peut restreindre l’accès aux Skype pour Business Server 2015 internes ou clients connectés via une connexion VPN ou de DirectAccess.</span><span class="sxs-lookup"><span data-stu-id="080c3-117">As a result, access to Skype for Business Server 2015 might be restricted to internal or clients connected through a VPN or DirectAccess connection.</span></span> 
  
- <span data-ttu-id="080c3-p107">**Protocole Digest** pour utilisateurs anonymes. L’authentification Digest n’est pas utilisée pour d’autres interactions clients.</span><span class="sxs-lookup"><span data-stu-id="080c3-p107">**Digest protocol** for so-called anonymous users. Digest authentication is not used for other client interactions.</span></span>
    
<span data-ttu-id="080c3-121">Skype pour l’authentification de Business Server 2015 consiste en deux phases :</span><span class="sxs-lookup"><span data-stu-id="080c3-121">Skype for Business Server 2015 authentication consists of two phases:</span></span>
  
1. <span data-ttu-id="080c3-122">Une association de sécurité est établie entre le client et le serveur.</span><span class="sxs-lookup"><span data-stu-id="080c3-122">A security association is established between the client and the server.</span></span>
    
2. <span data-ttu-id="080c3-p108">Le client et le serveur utilisent l’association de sécurité existante pour signer les messages qu’ils envoient et vérifier les messages qu’ils reçoivent. Les messages non authentifiés d’un client ne sont pas acceptés lorsque l’authentification est activée sur le serveur.</span><span class="sxs-lookup"><span data-stu-id="080c3-p108">The client and server use the existing security association to sign messages that they send and to verify the messages they receive. <span data-ttu-id="080c3-p109">La confiance de l’utilisateur est associée à chaque message qui provient d’un utilisateur, et non à l’identité de l’utilisateur. Le serveur vérifie chaque message à la recherche d’informations d’identification d’utilisateur valides. Si les informations sont valides, le message est accepté non seulement par le premier serveur qui le reçoit mais par tous les autres serveurs dans le cloud de serveurs approuvés.</span><span class="sxs-lookup"><span data-stu-id="080c3-p109">User trust is attached to each message that originates from a user, not to the user identity itself. The server checks each message for valid user credentials. If the user credentials are valid, the message is unchallenged not only by the first server to receive it but by all other servers in the trusted server cloud.</span></span>
  
<span data-ttu-id="080c3-128">Les utilisateurs disposant d’informations d’identification valides émises par un partenaire fédéré sont approuvés mais peuvent éventuellement subir des contraintes supplémentaires les empêchant de profiter de la gamme complète de privilèges accordés aux utilisateurs internes.</span><span class="sxs-lookup"><span data-stu-id="080c3-128">Users with valid credentials issued by a federated partner are trusted but optionally prevented by additional constraints from enjoying the full range of privileges accorded to internal users.</span></span>
  
<span data-ttu-id="080c3-129">Les protocoles ICE et TURN utilisent également le défi Digest tel que décrit dans le RFC TURN de l’IETF.</span><span class="sxs-lookup"><span data-stu-id="080c3-129">The ICE and TURN protocols also use the Digest challenge as described in the IETF TURN RFC.</span></span>
  
<span data-ttu-id="080c3-130">Les certificats clients fournissent un autre moyen pour les utilisateurs doivent être authentifiés par Skype pour Business Server 2015.</span><span class="sxs-lookup"><span data-stu-id="080c3-130">Client certificates provide an alternate way for users to be authenticated by Skype for Business Server 2015.</span></span> <span data-ttu-id="080c3-131">Au lieu de fournir un nom d’utilisateur et un mot de passe, les utilisateurs disposent d’un certificat et d’une clé privée correspondant au certificat requis pour résoudre un défi cryptographique.</span><span class="sxs-lookup"><span data-stu-id="080c3-131">Instead of providing a user name and password, users have a certificate and the private key corresponding to the certificate that is required to resolve a cryptographic challenge.</span></span> <span data-ttu-id="080c3-132">(Ce certificat doit avoir un nom d’objet ou d’un autre nom du sujet qui identifie l’utilisateur et doit être délivré par une autorité de certification racine qui est approuvée par les serveurs exécutant Skype pour Business Server 2015 et se situer dans la période de validité du certificat ne peut pas avoir été révoqué.) To be authenticated, users only need to type in a personal identification number (PIN).</span></span> <span data-ttu-id="080c3-133">Les certificats s’avèrent particulièrement utiles pour les téléphones, les téléphones mobiles et autres appareils où il est difficile de saisir un nom d’utilisateur et un mot de passe.</span><span class="sxs-lookup"><span data-stu-id="080c3-133">Certificates are particularly useful for telephones, mobile phones, and other devices where it is difficult to enter a user name and password.</span></span>
