# <a name="plan-your-enterprise-voice-solution-in-skype-for-business-server-2015"></a>Planifier votre solution de Voix Entreprise dans Skype Business Server 2015
 
**Résumé :** Obtenir des informations sur les options disponibles pour la planification d’une solution de voix et de communications unifiée dans Skype pour Business Server 2015.
  
Skype pour Business Server offre deux sur site options de Voix Entreprise : Voix Entreprise et appeler Via le travail — pour intégrer le système téléphonique de votre organisation avec votre Skype pour le déploiement du serveur de l’entreprise, création d’un véritable de communications unifiées solution. Les deux options permettent aux utilisateurs d’utiliser leur Skype pour client d’entreprise pour initier des appels vocaux avec d’autres utilisateurs, à l’intérieur et à l’extérieur de votre organisation.
  
Vous pouvez également tirer parti des services de PBX de nuage de Microsoft, y compris les services de PBX et les appels RTPC, en connectant votre infrastructure de téléphonie sur site pour les services fournis par Skype pour entreprise en ligne. Pour plus d’informations, consultez [planification de votre système téléphonique dans les solutions Office 365 (Cloud PBX)](../../skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-your-phone-system-cloud-pbx-solution.md).
  
Vous pouvez aussi utiliser conjointement les Voix Entreprise et appeler Via le travail.
  
- Voix Entreprise est la voix plus complète offrant. C’est une solution logicielle VoIP (Voice over IP), contenant une riche palette de fonctionnalités, notamment l’intégration étroite avec Outlook et Exchange, et de nombreuses autres fonctionnalités comme les services Response Group, le parcage d’appel, la prise d’appel de groupe, les appels d’équipe et la prise en charge des les services d’urgence (Enhanced Emergency E9-1-1). Voix Entreprise, les utilisateurs d’utiliser un périphérique audio tel qu’un casque d’écoute avec leur ordinateur, ou un téléphone PBX ou un téléphone VoIP, au lieu d’une réseau public commuté classique.
    
- Appeler Via le travail offre un moyen d’intégrer votre Skype pour solution professionnelle avec vos systèmes de téléphone PBX existants. Cliquez sur un utilisateur activé pour appeler Via le travail dans Skype pour entreprises à l’appel d’un autre utilisateur, soit au sein de votre déploiement ou d’un utilisateur externe. L’appel est effectué via le téléphone PBX de l’utilisateur. En outre, ces utilisateurs cliquez Skype pour les entreprises à participer à des conférences.
    
    Appeler Via le travail de déploiement est aussi un avantage pour les utilisateurs qui ont des téléphones PBX toujours mais qui sont en cours de transition de Voix Entreprise, car elle permet à ces utilisateurs d’en cours à l’aide de leur Skype pour client d’entreprise pour contrôler leurs appels téléphoniques.
    
     Appeler Via le travail propose un ensemble plus limité de voix thanEnterprise de fonctionnalités vocales. Par exemple, il ne gère pas délégation, appel d’équipe, groupe réponse ou amélioré E9-1-1.
    
Bien sûr, vous pouvez avoir certains de vos utilisateurs activés pour la Voix Entreprise tandis que les autres utilisateurs utilisent les téléphones PBX. En outre, pendant la transition vers une solution complète de Voix Entreprise, vous pouvez activer les utilisateurs dont les téléphones PBX pour Voix Entreprise. Ces utilisateurs peuvent utiliser leur téléphone PBX tandis que sur leur bureau et permet également de Voix Entreprise effectuer ou de recevoir des appels à l’aide de périphériques VoIP tandis que dans d’autres emplacements. Si ces utilisateurs sont activés pour appeler Via le travail, ils peuvent utiliser leur Skype pour client d’entreprise pour contrôler leur téléphone PBX tandis que sur leur bureau.
  
> [!NOTE]
> Contrôle d’appel distant était une fonctionnalité proposée dans les versions précédentes de Lync Server, permettant aux utilisateurs d’utiliser leur Skype pour client d’entreprise pour envoyer et recevoir des appels sur leur téléphone PBX. Appel distant contrôle n’est pas pris en charge pour les utilisateurs hébergés sur Skype pour les serveurs d’entreprise, mais il est pris en charge pour les utilisateurs avec Skype pour les clients d’entreprise qui sont toujours hébergées sur les serveurs exécutant Lync Server 2013.
