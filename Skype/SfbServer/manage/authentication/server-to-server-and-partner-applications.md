# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="7e9c0-103">Gérer l’authentification de serveur à serveur (OAuth) et les applications partenaires dans Skype entreprise Server</span><span class="sxs-lookup"><span data-stu-id="7e9c0-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>

<span data-ttu-id="7e9c0-105">Skype entreprise Server doit être en mesure de communiquer de manière sécurisée avec d’autres applications et produits serveurs.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="7e9c0-106">Par exemple, vous pouvez configurer Skype entreprise Server de manière à ce que les données de contact et/ou les données d’archivage soient stockées dans Microsoft Exchange Server 2013 ; Toutefois, cette opération n’est possible que si Skype entreprise Server et Exchange sont en mesure de communiquer de manière sécurisée.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="7e9c0-107">De même, vous pouvez planifier une conférence Skype entreprise Server à partir d’Office Web Apps Server ; là encore, cette opération est possible uniquement si les deux serveurs (SharePoint et Skype entreprise Server) s’approuvent mutuellement.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; <span data-ttu-id="7e9c0-109">L’utilisation d’une méthode standardisée unique pour l’authentification de serveur à serveur est l’approche adoptée par Skype entreprise Server.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="7e9c0-110">Démarré avec la version Office Server 2013, Skype entreprise Server (ainsi que d’autres produits serveur Microsoft, y compris Exchange Server et SharePoint Server) prenait en charge le protocole OAuth (Open Authorization) pour l’authentification et l’autorisation de serveur à serveur.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="7e9c0-111">Avec OAuth, un protocole d’autorisation standard utilisé par un certain nombre de sites principaux, les informations d’identification de l’utilisateur et les mots de passe ne sont pas transmis d’un ordinateur à un autre.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="7e9c0-112">Au lieu de cela, l’authentification et l’autorisation sont basées sur l’échange de jetons de sécurité ; Ces jetons accordent l’accès à un ensemble spécifique de ressources pour une durée spécifique.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="7e9c0-113">L’authentification OAuth implique généralement trois parties : un serveur d’autorisation unique et les deux domaines qui doivent communiquer les uns avec les autres.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="7e9c0-114">(Vous pouvez également effectuer l’authentification de serveur à serveur sans utiliser de serveur d’autorisation, un processus qui sera abordé plus loin dans ce document.) Les jetons de sécurité sont émis par le serveur d’autorisation (également appelé serveur de jetons de sécurité) pour les deux domaines qui doivent communiquer ; Ces jetons vérifient que les communications provenant d’un domaine doivent être approuvées par l’autre domaine.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) Security tokens are issued by the authorization server (also known as a security token server) to the two realms that need to communicate; these tokens verify that communications originating from one realm should be trusted by the other realm.</span></span> <span data-ttu-id="7e9c0-115">Par exemple, le serveur d’autorisation peut émettre des jetons qui vérifient que les utilisateurs d’un domaine Skype entreprise Server spécifique sont en mesure d’accéder à un domaine Exchange spécifié, et inversement.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-115">For example, the authorization server might issue tokens that verify that users from a specific Skype for Business Server realm are able to access a specified Exchange realm, and vice-versa.</span></span>
  
> [!NOTE]
> <span data-ttu-id="7e9c0-116">Un domaine est tout simplement un conteneur de sécurité.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-116">A realm is simply a security container.</span></span> <span data-ttu-id="7e9c0-117">Par défaut, Skype entreprise Server utilise votre domaine SIP par défaut comme domaine OAuth.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="7e9c0-118">Des espaces de noms SIP supplémentaires sont ajoutés à la liste autre nom de l’objet dans le certificat OAuth.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="7e9c0-119">Skype entreprise Server prend en charge trois scénarios d’authentification de serveur à serveur.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="7e9c0-120">Avec Skype entreprise Server, vous pouvez :</span><span class="sxs-lookup"><span data-stu-id="7e9c0-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="7e9c0-121">Configurez l’authentification de serveur à serveur entre une installation locale de Skype entreprise Server et une installation locale d’Exchange et/ou de SharePoint Server.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="7e9c0-122">Configurez l’authentification de serveur à serveur entre une paire de composants Microsoft 365 ou Office 365 (par exemple, entre Microsoft Exchange Server et Skype entreprise Server, ou entre Skype entreprise Server et SharePoint).</span><span class="sxs-lookup"><span data-stu-id="7e9c0-122">Configure server-to-server authentication between a pair of Microsoft 365 or Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="7e9c0-123">Configurez l’authentification de serveur à serveur dans un environnement intersite (c’est-à-dire, l’authentification de serveur à serveur entre un serveur local et un composant Microsoft 365 ou Office 365).</span><span class="sxs-lookup"><span data-stu-id="7e9c0-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and a Microsoft 365 or Office 365 component).</span></span>
    
<span data-ttu-id="7e9c0-124">Notez que, à ce stade, seuls Exchange 2013, SharePoint Server, Lync Server 2013, Skype entreprise Server 2015 et Skype entreprise 2019 prennent en charge l’authentification de serveur à serveur ; Si vous n’exécutez pas l’un de ces serveurs, vous ne pourrez pas totalement implémenter l’authentification OAuth.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="7e9c0-125">Il convient également de remarquer que l’authentification de serveur à serveur est facultative : si Skype entreprise Server n’a pas besoin de communiquer avec d’autres serveurs (par exemple, Exchange), l’authentification de serveur à serveur peut être ignorée.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="7e9c0-126">Si l’authentification de serveur à serveur est déjà configurée pour Lync Server 2013 et d’autres applications, il n’est pas nécessaire de la relancer pour Skype entreprise Server.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="7e9c0-127">Toutefois, l’authentification de serveur à serveur est requise si vous souhaitez utiliser certaines des fonctionnalités de Skype entreprise Server, telles que le magasin de contacts unifié.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="7e9c0-128">Avec le magasin de contacts unifié, les informations de contact de Skype entreprise Server sont stockées dans Exchange et non dans Skype entreprise Server ; <span data-ttu-id="7e9c0-131">Pour que la version Microsoft 365 ou Office 365 de Skype entreprise Server communique avec son homologue Exchange, Skype entreprise Server doit d’abord obtenir un jeton de sécurité auprès du serveur d’autorisation.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-131">For the Microsoft 365 or Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="7e9c0-132">Skype entreprise Server utilise ensuite ce jeton de sécurité pour s’identifier auprès d’Exchange.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="7e9c0-133">Les versions de Microsoft 365 ou Office 365 d’Exchange doivent passer par le même processus afin de communiquer avec Skype entreprise Server.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-133">The Microsoft 365 or Office 365 versions of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="7e9c0-134">Toutefois, en ce qui concerne l’authentification de serveur à serveur entre deux serveurs Microsoft, il n’y a aucun besoin d’utiliser un serveur de jetons tiers.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="7e9c0-135">Les produits serveur tels que Skype entreprise Server et Exchange disposent d’un serveur de jetons intégré qui peut être utilisé à des fins d’authentification avec d’autres serveurs Microsoft (par exemple, SharePoint Server) qui prennent en charge l’authentification de serveur à serveur.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="7e9c0-136">Par exemple, Skype entreprise Server peut émettre et signer un jeton de sécurité lui-même, puis utiliser ce jeton pour communiquer avec Exchange.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="7e9c0-137">Dans un cas comme celui-ci, aucun serveur de jeton tiers n'est nécessaire.</span><span class="sxs-lookup"><span data-stu-id="7e9c0-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="7e9c0-138">Pour configurer l’authentification de serveur à serveur pour une implémentation locale de Skype entreprise Server, vous devez effectuer les deux opérations suivantes :</span><span class="sxs-lookup"><span data-stu-id="7e9c0-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
