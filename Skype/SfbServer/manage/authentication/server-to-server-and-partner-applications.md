<span data-ttu-id="efe3a-105">Skype entreprise Server doit être en mesure de communiquer en toute sécurité avec d’autres applications et des produits serveur.</span><span class="sxs-lookup"><span data-stu-id="efe3a-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="efe3a-106">Par exemple, vous pouvez configurer Skype entreprise Server de façon à ce que les données de contact et/ou les données d’archivage soient stockées dans Microsoft Exchange Server 2013. > <span data-ttu-id="efe3a-116">Un domaine est tout simplement un conteneur de sécurité.</span><span class="sxs-lookup"><span data-stu-id="efe3a-116">A realm is simply a security container.</span></span> <span data-ttu-id="efe3a-117">Par défaut, Skype entreprise Server utilise votre domaine SIP par défaut comme domaine OAuth.</span><span class="sxs-lookup"><span data-stu-id="efe3a-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="efe3a-118">Des espaces de noms SIP supplémentaires sont ajoutés à la liste Autre nom du sujet du certificat dans le certificat OAuth.</span><span class="sxs-lookup"><span data-stu-id="efe3a-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span>

- <span data-ttu-id="efe3a-123">Configurer l’authentification de serveur à serveur dans un environnement local (c’est-à-dire, l’authentification de serveur à serveur entre un serveur local et un composant Office 365).</span><span class="sxs-lookup"><span data-stu-id="efe3a-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and an Office 365 component).</span></span>
