# <a name="sql-server-reporting-services-invoke"></a>SQL Server Reporting Services (Appel)
 
Après avoir fourni les informations requises pour le déploiement du serveur de surveillance des États à Microsoft SQL Server 2008 R2 ou à des Services de rapport Microsoft SQL Server 2012, la page exécuter de commandes affiche un résumé des commandes lancées pour installer la rapports sur SQL Server Reporting Services.
  
Passez en revue le résumé des commandes et notez les messages d’erreur ou d’avertissement affichés à partir des commandes. Si un fichier journal est généré, sélectionnez-le dans la liste déroulante sous la fenêtre du résumé, puis cliquez sur **Afficher le journal** pour afficher le fichier journal.
  
> [!IMPORTANT]
> Pour les rapports Reporting Services pour déployer avec succès et pour accéder aux rapports après le déploiement est terminé, vous devez avoir TCP/IP port 80 (et le cas échéant, le port TCP 443 pour SSL, si vous attribuez un certificat pour le Reporting Services) ouvrent dans le pare-feu Windows avec la sécurité avancée sur le SQL Server. Pour plus d’informations, voir [configurer le pare-feu Windows pour autoriser l’accès SQL Server](https://go.microsoft.com/fwlink/p/?linkId=218031) pour Microsoft SQL Server 2008 R2.
  
Après avoir passé en revue le résumé, cliquez sur **Terminer** pour terminer l’installation de le rapports SQL Server Reporting Services.
