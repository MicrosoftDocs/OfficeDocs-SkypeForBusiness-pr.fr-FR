# <a name="certificate-request-certificate-authority-account"></a>Demande de certificat (compte d’autorité de certification)
 
Pour l’envoi d’une demande, votre autorité de certification (CA) doit avoir besoin d’informations d’identification autres que celles de l’utilisateur pour lequel vous êtes actuellement connecté. Pour autoriser une demande de certificat en tant qu’utilisateur différent, activez la case à cocher **spécifier d’autres informations d’identification pour l’autorité de certification**, puis tapez le nom d’utilisateur ou le nom d’utilisateur du _domaine_\ _pour un_ utilisateur qui peut demander le certificat. Dans la zone de texte **mot de passe** , tapez le mot de passe de l’utilisateur que vous avez spécifié. Le nom d’utilisateur et le mot de passe sont ensuite envoyés dans le cadre du processus de demande de certificat, mais pas dans la demande de certification réelle.
