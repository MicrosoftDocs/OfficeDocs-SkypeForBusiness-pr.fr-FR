# <a name="publish-topology-create-mirror-database-page"></a>Publier la topologie : page Créer une base de données miroir
 
Cette documentation est préliminaire et sujette à modification. Des rubriques vides sont incluses comme espaces réservés.
  
Le générateur de topologies analyse tous les paramètres de configuration dans votre document de topologie défini. Si vous avez configuré les bases de données SQL Server mises en miroir, vous voyez un écran qui affiche le magasin miroir, le magasin principal, le chemin d’accès au partage de fichiers et les chemins d’accès à la base de données. Vous activez une case à cocher associée à un magasin miroir de base de données pour créer la base de données mise en miroir et le chemin d’accès de partage de fichiers. Vous désactivez une case à cocher pour un miroir de base de données si vous ne souhaitez pas créer et déployer la base de données sur le serveur SQL défini.
  
 Si vous choisissez de ne pas installer une ou toutes ces bases de données pour le moment, vous effacez les bases de données que vous ne pouvez pas installer. À une date ultérieure, vous choisirez d' **installer la base de données** en cliquant avec le bouton droit sur la plupart des éléments de nœud dans le générateur de topologie.
