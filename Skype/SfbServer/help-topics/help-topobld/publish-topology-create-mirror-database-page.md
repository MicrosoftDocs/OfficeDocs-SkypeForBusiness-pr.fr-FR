# <a name="publish-topology-create-mirror-database-page"></a>Publier la topologie de créer la Page de base de données mise en miroir
 
Cette rubrique fait partie de la documentation préliminaire. Elle est susceptible d’être modifiée lors des prochaines mises à jour. Les rubriques vides apparaissent sous la forme d’espaces réservés.
  
Générateur de topologies analyse tous les paramètres de configuration dans votre document de topologie définie. Si vous avez configuré des bases de données en miroir de SQL Server, un écran qui affiche la banque de miroir, la banque principale, chemin d’accès du partage de fichier et les chemins d’accès de la base de données s’affiche. Vous activez une case à cocher associée à une banque de mise en miroir de base de données pour créer la base de données en miroir et le chemin d’accès du partage de fichier. Vous désactivez une case à cocher pour une mise en miroir de base de données si vous ne souhaitez pas créer et déployer la base de données pour le SQL Server définie.
  
 Si vous sélectionnez pour choisir de ne pas installer une ou toutes ces bases de données pour l’instant, vous désactivez les bases de données que vous ne voulez pas installer. Par la suite, vous choisissez d’utiliser les **Installer la base de données** en cliquant avec le bouton droit sur la plupart des éléments d’un nœud dans le Générateur de topologie.
