# <a name="publish-topology-create-mirror-database-page"></a>Publier la topologie : page Créer une base de données miroir
 
Cette documentation est préliminaire et sujette à modification. Des rubriques vides sont incluses comme espaces réservés.
  
Le Générateur de topologies pare tous les paramètres de configuration dans votre document de topologie défini. Si vous avez configuré des bases de données SQL Server miroir, un écran affiche le magasin miroir, le magasin principal, le chemin d’accès au partage de fichiers et les chemins de base de données. Cochez une case associée à un magasin miroir de base de données pour créer la base de données en miroir et le chemin d’accès du partage de fichiers. Vous pouvez effacer une case à cocher pour un miroir de base de données si vous ne souhaitez pas créer et déployer la base de données sur le SQL Server.
  
 Si vous choisissez de ne pas installer une ou toutes ces bases de données pour le moment, vous devez effacer les bases de données que vous n’installerez pas. Plus tard, vous choisirez d’utiliser **Installer** la base de données en cliquant avec le bouton droit sur la plupart des éléments de nœud dans le Générateur de topologie.
