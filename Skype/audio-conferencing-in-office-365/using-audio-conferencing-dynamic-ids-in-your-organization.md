# Utilisation d'identificateurs dynamiques audioconférence dans votre organisation

> [!IMPORTANT]
> Cet article a été traduit automatiquement, voir l'avertissement.  
  
Le service de conférence Audio est mis à jour pour offrir chaque Skype entreprise et Teams Microsoft réunion identifiants de conférence différents. ID de conférence dynamique dépassent une amélioration significative de conférence statique ID, car ils permettent :
  
- **Sécurité améliorée** La conférence ID sont uniques pour chaque Skype entreprise ou Microsoft Teams réunion et est générées lors de la réunion.
    
- **Une meilleure expérience pour les réunions consécutives et simultanées :** des informations de rendez-vous spécifiques sont fournies aux réunions pour un même organisateur, ce qui empêche les participants téléphoniques à une réunion de s'ajouter aux participants d'une autre réunion lorsqu'elles sont prévues à peu de temps d'intervalle.
    
- **Une transition transparente :** lorsque votre organisation active les ID de conférence dynamiques, toutes les réunions qui ont déjà été planifiées dans votre organisation avec des ID de conférence statiques auront lieu.
    
> [!TIP]
> ID dynamiques est uniquement disponibles aux utilisateurs qui sont activées pour les ** Conférences Audio** et définie les Microsoft comme leur fournisseur de services d'audioconférence. Vous pouvez[Affecter Microsoft comme fournisseur de services d'audioconférence](assign-microsoft-as-the-audio-conferencing-provider.md) pour vos utilisateurs.
  
## Quelles sont les modifications verront les utilisateurs de mon organisation ?

Après l'activation des ID de conférence dynamique pour votre organisation, tout nouveau Skype entreprise ou Microsoft Teams réunion qui est planifiée par les utilisateurs de votre organisation qui sont activés pour audioconférence aura conférence ID sera différente de la ID de conférence statique qu'avant. Organisateurs qui disposaient statique ID de conférence avant utile de rappeler aux utilisateurs participation à des réunions dont ils ont besoin maintenant utiliser un nouvel ID de conférence dans l'invitation de la réunion avant de pouvoir le rejoindre.
  
> [!NOTE]
> Réunions qui ont été planifiées par un utilisateur avec des identifiants de conférence statique avant de l'organisation a été activée pour la conférence dynamique ID continuent à utiliser les codes de conférence statique, afin qu'ils allez continuer planifier des réunions sans impact.
