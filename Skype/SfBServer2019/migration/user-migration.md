# <a name="user-migration"></a>Migration des utilisateurs

Une meilleure pratique généralement acceptée pour les migrations consiste à créer plusieurs utilisateurs de test et de les utiliser pour effectuer des tests de systèmes. Une fois que vous avez déplacé et testé ces comptes, identifier un groupe d’utilisateurs du pilote de production et déplacer leurs comptes et effectuer des tests de validation sur eux. Lorsque vous recevez des résultats satisfaisants, vous pouvez déplacer le reste de vos utilisateurs vers le nouveau déploiement.
  
Pour plus d’informations sur l’activation des utilisateurs pour Skype pour Business Server 2019, voir le Skype Business Server 2019 documentation de déploiement.
