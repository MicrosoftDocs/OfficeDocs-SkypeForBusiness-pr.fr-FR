# Activation et à l'aide de tableau de bord qualité d'appel pour Microsoft Teams et Skype entreprise Online

Découvrez comment configurer votre organisation Office 365 pour utiliser le tableau de bord qualité d'appel pour contrôler la qualité des appels.
  
> [!IMPORTANT]
> Cet article a été traduit automatiquement, voir l'[avertissement](553fa13c-92d2-4d5c-a3d5-41a073cb047c.md#MT_Footer). Vous pouvez consulter la version en anglais de cet article [ici](https://support.office.com/en-us/article/553fa13c-92d2-4d5c-a3d5-41a073cb047c). 
  
Le tableau de bord de la qualité d'appel (CQD) pour Microsoft Teams et Skype Entreprise Online vous permet d'obtenir des informations sur la qualité des appels effectuées à l'aide de services Microsoft Teams et Skype Entreprise. Cette rubrique décrit les étapes que vous devez effectuer pour commencer à collecter des données.
  
> [!NOTE]
> Les rapports détaillés du tableau de bord de qualité des appels sont actuellement disponibles en version bêta mais sont disponibles pour tous les clients. 
  
## Dernières modifications et mises à jour

Les dernières modifications apportées à CQD sont les suivantes :
  
- Inclut les données Microsoft Teams en plus de données Skype Entreprise Online.
    
- Rapports de synthèse comprennent un filtre de produit pour sélectionner toutes les données, données Microsoft Teams ou Skype Entreprise Online données.
    
Consultez cet article pour obtenir une liste de [Dimensions et mesures disponibles dans appeler la qualité du tableau de bord pour Microsoft Teams et Skype entreprise Online](dimensions-and-measures-available-in-call-quality-dashboard-for-microsoft-teams.md).
  
> [!NOTE]
> Pour obtenir des informations sur les mises à jour et les modifications apportées au tableau de bord, cliquez sur le lien **Bonne nouvelle**. Vous pouvez accéder au [Tableau de bord de qualité des appels](https://aka.ms/CQDOnline). 
  
## Activation des rapports synthèse Microsoft appel qualité du tableau de bord (CQD)

Avant de démarrer à l'aide de la CQD, vous devez activer pour votre organisation Office 365.
  
1. Se connecter à votre organisation Office 365 à l'aide d'un compte d'administrateur et sélectionnez la vignette **administrateur** pour ouvrir le centre d'administration.
    
2. Dans le volet gauche, sous **Centre d'administration**, sélectionnez **Skype entreprise** pour ouvrir le centre d'administration Skype Entreprise.
    
3. Dans le centre d'administration Skype Entreprise, sélectionnez **Outils** dans le volet gauche, puis **Skype pour entreprise Online appeler la qualité du tableau de bord**.
    
     ![Skype for Business tools](../images/6cc7f80f-b8e2-4a9b-aab8-ac871d07a261.png)
  
4. Dans la page qui s'ouvre, ouvrez une session avec votre compte d'administrateur Global et puis fournissez les informations d'identification du compte lorsque vous y êtes invité.
    
     ![CQD Login](../images/ac4c1699-d8c1-4bda-af30-0fec35b5fd22.png)
  
Une fois que vous êtes connecté, une fois activé, la CQD commencera collecte et traitement des données.
  
> [!NOTE]
> Il peut prendre quelques heures pour traiter les données nécessaires pour afficher des résultats significatifs dans les rapports.

### Rapports prêts à l'emploi

Les deux éditions de CQD offrent une hors de la zone expérience, vous donnant un appel métriques de qualité sans avoir besoin de créer les nouveaux rapports. Une fois que les données sont traitées dans le serveur principal, vous pouvez commencer à voir les données de qualité d'appel dans les rapports.
  
### Présentation des rapports

Les deux éditions du tableau de bord de qualité des appels fournissent un point d'entrée de haut niveau pour surveiller les informations générales de qualité des appels, mais ces informations sont présentées de manière différente dans les rapports de synthèse et les rapports détaillés.
  
Les rapports de synthèse offrent un affichage simplifié sous forme de page à onglets, qui permet aux utilisateurs de parcourir et de comprendre rapidement l'état et les tendances de la qualité globale des appels.
  
Les quatre onglets sont les suivants :
  
- **La qualité des appels globale** - fournit des informations sur tous les flux, c'est-à-dire une agrégation de flux de données Client-serveur et flux Client Client, ainsi que distinct-Client et le serveur flux Client Client, dans le formulaire des tendances mensuels et quotidiens.
    
- **Serveur-client**: fournit des informations supplémentaires pour les flux transitant entre les points de terminaison serveur et client.
    
- **Client-client**: fournit des informations supplémentaires pour les flux transitant entre deux points de terminaison client.
    
- **SLA de qualité de voix** - fournit des informations relatives aux appels qui sont inclus dans le Skype Entreprise Online SLA de qualité de voix.
    
### Onglet Qualité globale des appels

Utiliser les données sous cet onglet pour évaluer la qualité de l'état de l'appel et les tendances en consultant les nombres de flux et les pourcentages médiocres. La légende dans le coin supérieur droit indique les couleurs et les éléments visuels représentent les mesures.
  
![CQD Data key](../images/c8d183b1-6592-49b0-a81d-35cc0568d5f0.png)
  
Flux de données est classés en trois groupes : bonne, mauvaise et non classés. Il sont calculées également  *un problème %*  des valeurs qui permettent de vous le rapport de flux de données considérés comme * un problème*  pour le nombre total de flux classés. Dans la mesure où *mauvaise % = flux mauvaise / (mauvaise flux + flux bonne) * 100*  , cela rend le *% mauvaise*  ne sont pas affectées par la présence avec plusieurs flux *Unclassified*  . Pour ce qui est utilisé pour classer un flux de données sous forme de bonne ou mauvaise, reportez-vous à[https://aka.ms/cqd_quality_thresholds](https://aka.ms/cqd_quality_thresholds).
  
Pour mesurer les valeurs de nombre d'appels, servez-vous de l'échelle de gauche.
  
![CQD data count](../images/850bd25d-d9b2-4df4-8ca6-526a528897c2.png)

### Onglet serveur Client et des onglets de Client Client

Ces deux onglets contiennent des détails supplémentaires sur les flux circulant dans leurs scénarios de point de terminaison à point de terminaison. Les deux onglets comportent quatre sections réductibles, représentant quatre scénarios selon lesquels les flux de données multimédias circuleraient.
  
- Connexion filaire interne
    
- Connexion filaire externe
    
- WiFi interne
    
- WiFi externe
    
#### Test interne

Pendant le traitement, la principale CQD considère un flux de données sous la forme  *à l'intérieur*  ou *à l'extérieur*  à l'aide des informations de construction, le cas échéant. Points de terminaison de chaque flux de données sont associés à une adresse. Si la constitue dans la liste des sous-réseaux dans les informations de construction téléchargées, il est considéré à l'intérieur. Si informations n'a pas encore été téléchargé, à l'intérieur de Test classera toujours les flux comme *à l'extérieur*  . Notez qu'à l'intérieur de Test pour le scénario de serveur Client considère uniquement le point de terminaison client. Les serveurs étant toujours en dehors du point de vue d'un utilisateur, il n'est pas pris en compte dans le test.
  
#### Wired comparatif Wi-Fi

Comme l'indique le nom, il s'agit d'un critère de classement en fonction du type de connexion client. Là encore, le serveur étant toujours connecté en filaire, il n'est pas inclus dans le calcul.
  
> [!NOTE]
> Étant donné un flux de données, si un des deux points de terminaison est connecté à un réseau Wi-Fi, puis il est classé comme Wi-Fi dans CQD.

## Transfert des informations relatives au bâtiment
<a name="BKMK_FeaturesOfTheCQD"> </a>

Le tableau de bord CQD résumé rapports comprend une page de **Téléchargement des données client**, accédée en sélectionnant le **Téléchargement des données client** dans le menu Paramètres dans le coin supérieur droit. Cette page est utilisée pour les administrateurs à télécharger leurs informations, par exemple de mappage d'adresse IP et informations géographiques, mappage de chaque point d'accès sans fil et son adresse MAC, etc..
  
![CQD Dashboard](../images/839c9ab4-0246-46c9-8402-aafd83a0bc63.png)
  
1. Dans la page de **Téléchargement des données client**, utilisez le menu déroulant pour choisir un type de fichier de données pour le téléchargement. Le type de données fichier désigne le contenu du fichier (par exemple, « Créer » fait référence au mappage d'adresse IP et la création d'ainsi que d'autres informations géographiques). Actuellement nous sommes uniquement en charge le type de données « Construction ». Quelques autres types de données soient ajoutés dans les versions ultérieures.
    
2. Après avoir sélectionné le type de données de fichier, cliquez sur **Parcourir** pour choisir un fichier de données.
    
  - Le fichier de données doit être un fichier .tsv (valeurs séparées par des tabulations) ou .csv (valeurs séparées par des virgules). Si vous utilisez un fichier .csv, n'importe quel champ qui contient une virgule doit contenir des devis ou que la virgule supprimée. Par exemple, si votre nom de construction est NY, NY, dans le fichier .csv elle doit être entrée en tant que « NY, NY ».
    
  - La taille du fichier de données ne doit pas dépasser 50 Mo.
    
  - Pour chaque fichier de données, les différentes colonnes du fichier doivent correspondre à un type de données prédéfini. Ce point sera abordé dans la suite de cette rubrique.
    
3. Après avoir sélectionné un fichier de données, spécifiez la **date de début** et, si vous le souhaitez, **Spécifiez une date de fin**.
    
4. Après avoir sélectionné la **date de début**, sélectionnez **Transférer** pour transférer le fichier sur le serveur de tableau de bord de qualité des appels.
    
    Avant que le fichier est téléchargé, il est tout d'abord validé. Une fois validée, il est stocké dans un blob Azure. Si validation échoue ou que le fichier ne peut pas être stockés dans un blob Azure, un message d'erreur s'affiche et demande une correction au fichier. L'image suivante montre un message d'erreur ne se produise lorsque le nombre de colonnes dans le fichier de données est incorrect.
    
     ![CQD Example upload validation error](../images/22716a32-3d3d-4870-983c-46089e8b212a.png)
  
5. En l'absence d'erreur lors de la validation, le transfert du fichier aboutit. Vous découvrez alors le fichier de données transféré dans le tableau **Mes transferts**, qui affiche la liste complète de tous les fichiers transférés pour le client actuel dans la partie inférieure de cette page.
    
    Chaque enregistrement indique client de télécharger un fichier de données, avec le type de fichier, la dernière mise à jour, période, description, supprimer et une icône de téléchargement. Pour supprimer un fichier, sélectionnez l'icône de la Corbeille de la Corbeille dans le tableau. Pour télécharger un fichier, sélectionnez l'icône de téléchargement dans la colonne de **téléchargement** de la table.
    
     ![CQD My Uploads table](../images/4168a883-bbea-461a-80b1-42eedf2e7732.png)
  
### Format du fichier de données client et structure du fichier de données du bâtiment
<a name="BKMK_TenantDataFile"> </a>

Pour que le format du fichier de données réussisse le contrôle de validation avant transfert, il doit respecter les critères ci-dessous.
  
- Le fichier doit être un fichier .tsv, ce qui signifie que, dans chaque ligne, les colonnes sont séparées par une tabulation, ou un fichier .csv avec chaque colonne séparée par une virgule.
    
- Le contenu du fichier de données n'inclut pas les en-têtes de tableau. Que signifie que la première ligne du fichier de données doit être des données réelles, sans les en-têtes comme « Réseau », etc..
    
- Pour chaque colonne, le type de données peut uniquement être chaîne, nombre ou Bool. Si c'est le nombre, la valeur doit être une valeur numérique ; s'il s'agit Bool, la valeur doit être 0 ou 1.
    
- Pour chaque colonne, si le type de données est chaîne, les données ne peuvent être vides (mais toujours doivent être séparés par un séparateur : approprié (c'est-à-dire, un onglet ou une virgule). Il affecte uniquement ce champ une valeur de chaîne vide.
    
- Il doit y avoir 14 colonnes pour chaque ligne, et chaque colonne doit correspondre au type de données suivant. De même, les colonnes doivent être dans l'ordre indiqué dans le tableau suivant :
    
|**Nom de colonne**|**Type de données**|**Exemple**|
|:-----|:-----|:-----|
|Réseau  <br/> |Chaîne  <br/> |192.168.1.0  <br/> |
|Nom réseau  <br/> |Chaîne  <br/> |USA/Seattle/SEATTLE-SEA-1  <br/> |
|Plage du réseau  <br/> |Nombre  <br/> |26  <br/> |
|Nom du bâtiment  <br/> |Chaîne  <br/> |SEATTLE-SEA-1  <br/> |
|Type de propriété  <br/> |Chaîne  <br/> |Contoso  <br/> |
|Type de bâtiment  <br/> |Chaîne  <br/> |Terminaison informatique  <br/> |
|Type de bureau dans le bâtiment  <br/> |Chaîne  <br/> |Ingénierie  <br/> |
|Ville  <br/> |Chaîne  <br/> |Seattle  <br/> |
|Code postal  <br/> |Chaîne  <br/> |98001  <br/> |
|Pays  <br/> |Chaîne  <br/> |États-Unis  <br/> |
|État  <br/> |Chaîne  <br/> |WA  <br/> |
|Région  <br/> |Chaîne  <br/> |MSUS  <br/> |
|Dans l'entreprise  <br/> |Booléen  <br/> |1  <br/> |
|ExpressRoute  <br/> |Booléen  <br/> |0  <br/> |
   
> [!IMPORTANT]
> La plage réseau peut être utilisée pour représenter un sur-réseau (une combinaison de plusieurs sous-réseaux avec un préfixe de routage unique). Tous les téléchargements de l'immeuble seront contrôlés afin de vérifier si des plages se chevauchent. Si vous avez déjà chargé un fichier du bâtiment, vous devez télécharger le fichier actuel et le charger de nouveau afin d'identifier les chevauchements éventuels et de corriger le problème avant de le charger de nouveau. Tout chevauchement dans les fichiers chargés auparavant peut entraîner le mappage incorrect des sous-réseaux vers les immeubles dans les rapports. > Certaines mises en œuvre VPN n'indiquent pas précisément les informations de sous-réseau. Il est recommandé que lorsque vous ajoutez un sous-réseau VPN à la création du fichier, au lieu d'une seule entrée pour le sous-réseau, séparez les entrées sont ajoutées pour chaque adresse dans le sous-réseau VPN comme un réseau 32 bits distinct. Chaque ligne peut avoir les mêmes métadonnées de construction. Par exemple, au lieu d'une seule ligne pour 172.16.18.0/24, vous devez disposer de 256 lignes, avec une seule ligne pour chaque adresse entre 172.16.18.0/32 et 172.16.18.255/32, inclus.
