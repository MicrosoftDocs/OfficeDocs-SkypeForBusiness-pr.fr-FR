# <a name="microsoft-staffhub-to-be-retired"></a>Fin de parcours pour Microsoft StaffHub

Effet 2019, octobre 1, Microsoft StaffHub sera être retirée. Nous créons StaffHub fonctionnalités dans Microsoft Teams. Aujourd'hui, les équipes inclut l’application des équipes de gestion de la planification et des fonctionnalités supplémentaires seront intégrées au fil du temps.

Ces modifications font partie des efforts pour permettre aux employés avec Microsoft 365 que nous. Avec des capacités pour les travailleurs Firstline dans des équipes de chaque employé de votre organisation seront en mesure d’utiliser des équipes pour simplifier leur journée de travail, de collaborer avec vos collègues et accéder aux informations et des connaissances pour leur permettre d’effectuer leur travail meilleures.

## <a name="frequently-asked-questions"></a>Forum aux questions

### <a name="about-staffhub-retirement"></a>À propos de retraite StaffHub

#### <a name="when-will-staffhub-be-retired"></a>Lorsque StaffHub être retirée ?

À partir du milieu de 2019 avril, Microsoft StaffHub ne sera plus disponible pour les nouveaux clients et vous ne sera plus en mesure d’accéder à l’application depuis le lancement d’application Office 365. Les utilisateurs en cours sera toujours en mesure d’utiliser le service jusqu'à octobre 1, 2019, fonctionnalités toutefois ne supplémentaires seront disponibles.

#### <a name="what-will-happen-once-staffhub-is-retired"></a>Que se passe-t-il lorsque StaffHub est obsolète ?

Microsoft StaffHub cessera de fonctionner pour tous les utilisateurs sur le 1 octobre 2019. Toute personne qui essaie d’ouvrir l’application s’affichera un message pronom pour télécharger les équipes. Si l’utilisateur possède une licence active qui inclut des équipes et l’organisation a effectué les étapes appropriées pour activer les équipes, leurs données et les fonctionnalités principales de StaffHub seront disponibles pour les lors de leur transition vers les équipes.

Entre avril 2019 et octobre 2019, StaffHub d’utilisateurs recevront les encourager les notifications dans l’application à utiliser des équipes.

#### <a name="will-teams-offer-all-the-functionality-currently-offered-in-staffhub"></a>Les équipes offre toutes les fonctionnalités actuellement proposées dans StaffHub ?

Dans 2019 janvier, équipes ont commencé à proposer des équipes qui a été précédemment disponibles dans Microsoft StaffHub. Cette fonctionnalité permet aux gestionnaires planifier et distribuer des planifications de travail d’équipe et la possibilité de passer en revue son calendrier et de modifier la planification des demandes à partir de leurs appareils mobiles de membres de l’équipe. Pour plus d’informations, voir [gérer l’application des équipes de votre organisation dans les équipes](manage-the-shifts-app-for-your-organization-in-teams.md).

Nous sommes engagés à proposer une expérience utilisateur de qualité et les fonctionnalités uniques de StaffHub à équipes. Pour plus d’informations sur lequel fonctionnalités seront disponibles dans les équipes, voir [fonctionnalités de travail Firstline bientôt disponible pour les équipes](firstline-worker-features-coming-to-teams.md).

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Que dois-je faire pour préparer pour que cette modification ?

Pour savoir comment préparer pour la transition à partir de StaffHub équipes dans les équipes, accédez à [préparer pour déplacer vos équipes StaffHub équipes](move-staffhub-teams-to-shifts-in-teams.md#prepare).

#### <a name="why-cant-users-in-my-organization-access-their-staffhub-team-from-shifts-in-teams"></a>Pourquoi les utilisateurs de mon organisation ne peut pas accéder leur équipe StaffHub d’équipes dans les équipes ?

Les équipes qui ont été créées avec StaffHub uniquement peuvent être visualisés et gérés dans StaffHub. Après un StaffHub équipe est déplacé vers les équipes, les utilisateurs seront en mesure d’afficher leurs prévisions de déplacements. Nous vous invitons à communiquer ce changement au début et à vous familiariser les utilisateurs avec des équipes.

### <a name="about-shifts"></a>À propos des équipes

#### <a name="what-is-shifts"></a>Nouveautés des équipes

L’application d’équipes dans les équipes est un outil de gestion de planification qui conserve les travailleurs Firstline connecté et synchronisation. Responsables de créer, mettre à jour et gérer les planifications de travail d’équipe pour les équipes et membres de l’équipe peuvent afficher leurs planifications, voir qui d’autre est prévue pour le jour, demande d’échange ou de proposer une équipe et demandes de congé, à partir de leurs appareils mobiles.  

#### <a name="which-plans-is-shifts-available-in"></a>Qui planifie est disponible dans les équipes ?

Déplacements est disponible sans coût supplémentaire des plans de composant d’équipes dans les éléments suivants :

- Office 365 Business Premium 
- Office 365 entreprise F1, E5 E1, E3,
- Office 365 pour l’éducation
- Microsoft 365 F1, E1, E3, E5

Pour plus d’informations, voir [Disponibilité d’équipes](manage-the-shifts-app-for-your-organization-in-teams.md#availability-of-shifts).

#### <a name="how-do-i-get-shifts"></a>Comment obtenir les équipes ? 

Si vous êtes déjà sur un plan incluant des équipes, vous pouvez démarrer à l’aide des équipes aujourd'hui dans le cadre d’équipes. Consultez la rubrique [mise en route d’équipes](https://support.office.com/en-us/article/get-started-in-shifts-5f3e30d8-1821-4904-be26-c3cd25a497d6).

#### <a name="which-devices-or-platforms-is-shifts-available-on"></a>Les appareils ou les plateformes est disponible sur équipes ?

Déplacements est disponible sur les équipes client web, client de bureau équipes et les clients mobiles équipes (iOS et Android).
 
#### <a name="are-there-differences-between-the-shifts-desktop-and-web-apps-and-the-mobile-ios-and-android-apps"></a>Y a-t-il des différences entre les applications web et de bureau de déplacements et les applications mobile (iOS et Android) ?

Pour les responsables, les applications web et de bureau optimisées pour créer des équipes, ajouter des membres, créer et publier des planifications et partager du contenu avec l’équipe. Responsables généralement utilisent l’application mobile pour approuver les demandes, envoyer des messages rapides vers l’équipe, appeler un membre d’équipe et collaborer en déplacement. 

Membres d’équipe, l’application mobile est optimisée pour afficher son calendrier, voir qui ils collaborent avec un jour, d’horloge et se déconnecter, définir la disponibilité, durée hors de la demande, d’échange ou proposent des équipes et ouvrir des équipes. Il existe des différences dans les utilisateurs peuvent faire dans les applications web et de bureau et les applications mobiles. Nous sommes en train de parité des fonctionnalités.

#### <a name="are-there-differences-between-the-shifts-ios-app-and-the-android-app"></a>Y a-t-il des différences entre l’application iOS équipes et l’application d’Android ?

Fonctionnalité principale est identique entre les équipes iOS et Android applications. Il existe des différences de l’interface utilisateur mineures entre les deux applications à aligner sur les modèles de conception recommandé.

#### <a name="what-browsers-does-shifts-work-on"></a>Sur les navigateurs équipes fonctionne-t-il ?

Équipes fonctionne sur tous les navigateurs qui prend en charge les équipes qui inclut les dernières versions de Mozilla Firefox, Google Chrome, Microsoft Edge et Internet Explorer. Pour plus d’informations, voir [obtenir des clients pour les équipes](../../get-clients.md).

#### <a name="what-languages-does-shifts-support"></a>Quelles sont les langues prises en charge par équipes ?

Équipes est disponible dans les langues suivantes : chinois simplifié, français, espagnol, russe, japonais, Français, portugais-portugais (Brésil), allemand, coréen, italien, chinois traditionnel, néerlandais, turc, suédois, danois et norvégien. Nous envisageons ajouter la prise en charge de langues supplémentaires à l’avenir.  

#### <a name="where-is-shifts-data-stored"></a>Où sont stockées les données de déplacements ?

Données de déplacements sont stockées dans Azure dans les centres de données en Amérique du Nord et en Europe. Contenu SharePoint tire parti des groupes d’Office 365 et utilise le même stockage et prise en charge.

Pour plus d’informations, voir [data de l’emplacement d’équipes](manage-the-shifts-app-for-your-organization-in-teams.md#location-of-shifts-data).

#### <a name="where-can-i-learn-more-about-shifts"></a>Où puis-je obtenir plus d’informations sur les équipes ?

Pour plus d’informations, consultez [https://products.office.com/microsoft-teams/staff-scheduling-software](https://products.office.com/microsoft-teams/staff-scheduling-software).

Pour obtenir des conseils d’administration, consultez la rubrique [gérer l’application des équipes de votre organisation dans les équipes](manage-the-shifts-app-for-your-organization-in-teams.md)

Pour obtenir de l’utilisateur final, accédez à [Équipes aide pour les travailleurs Firstline](https://support.office.com/article/apps-and-services-cc1fba57-9900-4634-8306-2360a40c665b).

### <a name="switch-from-staffhub-to-shifts-in-teams"></a>Passer de StaffHub à des équipes de travail en équipe

#### <a name="what-licenses-do-i-need-for-teams"></a>De quelles licences ai-je besoin pour les équipes ?

Chaque utilisateur doit disposer d’une licence Microsoft 365 ou Office 365 active à partir d’un [plan éligible](#which-plans-is-shifts-available-in) et doit être attribué une licence d’équipes. Pour plus d’informations, voir [déplacer vos équipes StaffHub équipes de travail en équipe](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-move-my-organizations-staffhub-teams-to-teams"></a>Comment déplacer des équipes de StaffHub de mon organisation aux équipes ?

Pour obtenir des instructions détaillées sur la façon d’effectuer la transition à partir de StaffHub aux équipes, voir [déplacer vos équipes Microsoft StaffHub pour les déplacements dans les équipes](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-enable-teams-in-my-organization"></a>Comment activer des équipes au sein de mon organisation ?

Par défaut, les équipes est activée pour toutes les organisations. Vous affectez des équipes licences aux utilisateurs pour contrôler l’accès aux équipes individuel. Pour plus d’informations, voir [Activer les équipes de votre organisation](../../Office-365-set-up.md) et de [gérer l’accès utilisateur aux équipes](../../user-access.md).

#### <a name="what-if-i-currently-have-skype-for-business-enabled-in-my-organization-but-not-teams"></a>Que se passe-t-il si j’ai actuellement Skype pour les entreprises activé dans mon organisation, mais pas les équipes ?

Les équipes prend en charge la coexistence avec Skype pour les entreprises. Pour plus d’informations, voir [comprendre les équipes et Skype pour l’interopérabilité et coexistence d’entreprise](../../teams-and-skypeforbusiness-coexistence-and-interoperability.md) et [Migration et l’interopérabilité pour les organisations à l’aide des équipes avec Skype pour les entreprises](../../migration-interop-guidance-for-teams-with-skype.md).

#### <a name="does-each-user-need-to-have-an-office-365-account-or-azure-active-directory-azure-ad-account"></a>Chaque utilisateur a-t-il besoin d’avoir un compte Office 365 ou un compte Azure Active Directory (AD Azure) ?

Oui, chaque responsable et membre d’équipe à l’aide des équipes et équipes au sein des équipes doit posséder un compte Office 365 (Azure AD) et doit être attribué une licence valide.
 
#### <a name="in-staffhub-i-could-add-a-special-prefix-to-all-office-365-groups-that-were-created-by-staffhub-teams-can-i-do-the-same-in-teams"></a>Dans StaffHub, je peux ajouter un préfixe spécial à tous les groupes de 365 Office qui ont été créés par les équipes StaffHub. Puis-je effectuer la même dans les équipes ?

Vous n’aurez la même fonctionnalité lorsque vous déplacez de StaffHub aux équipes. Si vous avez besoin d’appliquer la cohérence des noms à tous vos groupes d’Office 365, vous pouvez utiliser la [stratégie de noms de groupes d’Office 365](https://docs.microsoft.com/office365/admin/create-groups/groups-naming-policy).

#### <a name="will-team-names-be-the-same-in-teams"></a>Noms d’équipe sera la même dans les équipes ?

Il est possible que le nom d’une équipe sera différent. Les noms d’équipe dans les équipes utilisent le nom de groupe dans Office 365. Si le nom de l’équipe StaffHub est différent du nom du groupe correspondant Office 365, vous verrez un nom différent lorsque vous basculez les équipes.

#### <a name="what-happens-to-team-membership-when-a-staffhub-team-is-moved-to-teams"></a>Que se passe-t-il à l’appartenance de l’équipe lorsqu’une équipe StaffHub est déplacée vers les équipes ?

L’appartenance de l’équipe est conservé lorsqu’une équipe StaffHub est déplacée vers les équipes. Numéros de téléphone des membres de l’équipe ne s’affiche dans les équipes. Si le numéro de téléphone StaffHub correspond au numéro de téléphone dans Azure AD, il sera stocké dans Azure AD pour ce compte d’utilisateur. 

Nous vous invitons à consulter vos règles de l’équipe existante sur StaffHub et si possible aligner les numéros de téléphone pour Azure pertinent identités AD.
 
#### <a name="what-happens-to-staffhub-team-members-who-arent-provisioned-or-whose-status-is-inactive"></a>Que se passe-t-il pour les membres de l’équipe StaffHub qui ne sont pas mis en service ou dont l’état est inactif ?

Planification de l’utilisateur est déplacée vers des équipes, toutefois, l’utilisateur n’est pas un membre de l’équipe ou un groupe d’Office 365.

Pour les comptes inactifs qui ne sont pas liés à un compte d’utilisateur AD Azure, un responsable peut faire en sorte l’utilisateur est ajouté en tant que membre de l’équipe en ajoutant une adresse de messagerie Office 365 valide dans la page de membres de l’équipe StaffHub afin que l’utilisateur est liée à Azure AD correct de compte dans le organisation infrarouge.

Une autre solution consiste pour les administrateurs pour exécuter l' **Add-StaffHubMember** et sauvegarder des applets de commande **Remove-StaffHubUser** pour supprimer un compte non mis en service d’une équipe StaffHub et ajouter l’utilisateur à l’équipe avec leur UPN.

Nous vous recommandons de vérifier que tous les utilisateurs de votre organisation sont abordées avec une licence appropriée afin de réduire l’interruption et l’accès aux services.
 
#### <a name="what-happens-to-chats-in-staffhub"></a>Que se passe-t-il à des conversations dans StaffHub ?

Pour vous aider à migrer les utilisateurs, nous planifier la migration de l’historique des conversations de données dans StaffHub aux équipes. La quantité de l’historique des conversations qui seront disponibles dans les équipes dépend de combien de temps de la configuration de votre organisation pour la transition. Nous vous recommandons de ne plus tard le 30 juin 2019 pour fournir suffisamment de temps pour une transition de 90 jours de données par le 1 octobre 2019 rien.

Assurez-vous que la création de groupe dans Office 365 est activée dans le centre d’administration Microsoft 365 avant le début de la transition, comme cela est essentiel pour nous la possibilité de déplacer vos données de conversation StaffHub dans les équipes. Pour activer ce service par le 30 juin 2019, provoque dans une date ultérieure de la migration des données de conversation en cours de copie sur maximum.

#### <a name="can-i-export-shift-schedules-from-shifts"></a>Puis-je exporter les planifications de travail d’équipe à partir des équipes ?
  
Vous pouvez exporter les informations de planification de travail d’équipe pour un fichier Excel à partir d’applications web et de bureau équipes. Vous pouvez ensuite importer ces informations aux systèmes et les outils de votre organisation.

#### <a name="how-can-i-recover-a-schedule-if-i-accidentally-deleted-a-team"></a>Comment puis-je récupérer une planification si vous avez supprimé une équipe ?

Suppression d’une planification est liée à la suppression d’une équipe. Si vous restaurez une équipe, la planification n’est pas récupérée.

#### <a name="what-powers-the-file-sharing-capability-in-teams-where-are-files-stored"></a>Le fichier de fonctionnalité dans les équipes de partage moteur ? Où sont stockés les fichiers ?

Le fichier de fonctionnalité, la capacité d’un responsable distribuer les fichiers aux membres d’équipe, de partage a été générée sur le serveur principal par SharePoint Online. Lors de la création d’une équipe, un groupe d’Office 365 est automatiquement mis en service pour que l’équipe en arrière-plan.

Pour plus d’informations, voir [comment SharePoint Online et OneDrive entreprise interagissent avec les équipes](../../SharePoint-OneDrive-interact.md).
 
#### <a name="is-there-integration-between-shifts-and-the-outlook-calendar-can-i-export-a-schedule-to-a-calendar"></a>Existe-t-il une intégration entre équipes et le calendrier Outlook ? Puis-je exporter un calendrier à un calendrier ?
 
Firstline employés peuvent afficher son calendrier de travail entière en un coup de œil dans les équipes. Nous allons en charge une intégration native avec le calendrier Outlook dans le cadre de notre feuille de route. En attendant, utilisez Flow Microsoft pour configurer l’intégration avec des équipes dans les équipes et Outlook et Google calendriers.

#### <a name="do-i-need-to-train-users-on-how-to-use-shifts-if-theyre-already-familiar-with-staffhub"></a>Ai-je besoin de former les utilisateurs sur l’utilisation des équipes s’ils sont déjà familiers avec StaffHub ?

Aucune formation supplémentaire n’est requise. Ils peuvent utiliser des équipes de façon similaire à la façon dont ils utilisé StaffHub. 
 
#### <a name="what-admin-controls-are-available-to-manage-whether-users-in-my-organization-can-use-shifts"></a>Les contrôles d’administration sont disponibles pour indiquer si les utilisateurs de mon organisation peuvent utiliser des équipes ?  
  
Vous pouvez activer ou désactiver des équipes org à l’échelle pour tous les utilisateurs dans votre organisation. Dans le centre d’administration Microsoft Teams, vous pouvez également utiliser la stratégie du programme d’installation d’application travailleur Firstline pour personnaliser les équipes de votre personnel Firstline. La stratégie est configurée à l’activité, équipes, la conversation et appel des applications. Affectation de ces applications les codes confidentiels de stratégie à la barre d’application dans les équipes pour accéder rapidement et facilement par les utilisateurs. 

Pour plus d’informations, voir [gérer l’application des équipes de votre organisation dans les équipes](manage-the-shifts-app-for-your-organization-in-teams.md).

#### <a name="what-powershell-controls-are-available-for-teams"></a>Les contrôles PowerShell sont disponibles pour les équipes ?

Vous gérez les équipes via le module PowerShell d’équipes Microsoft et la Skype pour le module PowerShell Business. Pour plus d’informations, voir [Vue d’ensemble de PowerShell équipes](../../teams-powershell-overview.md). À ce stade, il n’existe aucune applets de commande PowerShell spécifiques disponibles pour gérer les déplacements.  

S’il s’agit de quelque chose que qui vous intéressent, nous envoyer une demande via le [portail de commentaires équipes sur UserVoice](https://microsoftteams.uservoice.com/forums/555103-public-preview/category/182881-developer-platform).

#### <a name="what-will-happen-with-the-staffhub-api-which-is-currently-in-private-preview"></a>Que se passe-t-il avec l’API StaffHub qui est actuellement en mode Aperçu privé ?

Pour plus d’informations, voir [utilisation de l’API de graphique Microsoft pour travailler avec les équipes](https://docs.microsoft.com/graph/api/resources/teams-api-overview).

#### <a name="is-reporting-available-to-view-shifts-usage-across-my-organization"></a>Un rapport est disponible pour l’affichage de l’utilisation des équipes au sein de mon organisation ?

Équipes étant une application dans les équipes, vous pouvez afficher l’utilisation par le biais de rapports d’équipes. Pour plus d’informations, consultez les [équipes de création de rapports dans le centre d’administration Microsoft équipes](../../teams-analytics-and-reports/teams-reporting-reference.md) et [rapports d’activité équipes dans le centre d’administration Microsoft 365](../../teams-activity-reports.md).
