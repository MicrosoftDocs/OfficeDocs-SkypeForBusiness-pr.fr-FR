## <a name="unclassified-streams"></a><span data-ttu-id="1a928-244">Flux non classifiés</span><span class="sxs-lookup"><span data-stu-id="1a928-244">Unclassified Streams</span></span>

<span data-ttu-id="1a928-245">Dans le tableau de performance  général, un flux est marqué comme non classé en cas d’échec de la connectivité ICE (Interactive Connectivity Connectivity) ou lorsque toutes les mesures requises pour calculer la classification des flux ne sont pas signalées.</span><span class="sxs-lookup"><span data-stu-id="1a928-245">In CQD, a stream is marked _Unclassified_ when Interactive Connectivity Establishment (ICE) connectivity fails or when all the metrics required to compute the stream classification are not reported.</span></span>

<span data-ttu-id="1a928-246">Pour vérifier les échecs de la connectivité ICE, examinez les dimensions « First Connectivity Ice » et « Second Connectivity Ice » pour une valeur « FAILED ».</span><span class="sxs-lookup"><span data-stu-id="1a928-246">To check for ICE connectivity failures, examine the dimensions "First Connectivity Ice" and "Second Connectivity Ice" for a "FAILED" value.</span></span> <span data-ttu-id="1a928-247">Si l’une ou l’autre des valeurs indique un échec, le flux est marqué comme _non classé._</span><span class="sxs-lookup"><span data-stu-id="1a928-247">If either value indicates a failure, the stream is marked as _Unclassified_.</span></span>

<span data-ttu-id="1a928-248">Si la connectivité ICE  a réussi pour un flux non classé, le flux est probablement considéré comme _non_ classé, car les métriques de flux clés n’ont pas été signalées.</span><span class="sxs-lookup"><span data-stu-id="1a928-248">If ICE connectivity succeeded for an _Unclassified_ stream, the stream is likely considered _Unclassified_ because key stream metrics were not reported.</span></span> <span data-ttu-id="1a928-249">Il existe plusieurs raisons pour lesquelles ces métriques ne peuvent pas être rapportées :</span><span class="sxs-lookup"><span data-stu-id="1a928-249">There are a few reasons these metrics may not be reported:</span></span>

- <span data-ttu-id="1a928-250">**Aucun rapport QoE n’a** été reçu. Les mesures utilisées pour la classification sont signalées dans un rapport QoE envoyé à la fin d’un appel.</span><span class="sxs-lookup"><span data-stu-id="1a928-250">**QoE reports were not received** — The metrics used for classification are reported in a QoE report sent at the end of a call.</span></span> <span data-ttu-id="1a928-251">Si ce rapport n’est pas produit (par exemple, parce que certains points de terminaison tiers peuvent ne pas envoyer QoE) ou ne peuvent pas être envoyés (par exemple, en raison d’une panne du réseau), le CQD ne peut pas classifier le flux.</span><span class="sxs-lookup"><span data-stu-id="1a928-251">If this report is not produced (for example, because some third-party endpoints may not send QoE) or could not be sent (for example, because of a network outage), CQD is unable to classify the stream.</span></span>

  > <span data-ttu-id="1a928-264">La dimension « Utilisation des paquets » et la mesure « Utilisation moyenne des paquets » peuvent être utilisées pour déterminer l’activité des paquets d’un flux.</span><span class="sxs-lookup"><span data-stu-id="1a928-264">The "Packet Utilization" dimension and "Avg Packet Utilization" measurement can be used to determine the packet activity of a stream.</span></span>
