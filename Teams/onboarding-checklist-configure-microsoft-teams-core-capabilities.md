# <a name="configure-microsoft-teams-core-capabilities"></a>Configurer les fonctionnalités de base de Microsoft Teams

| Non | Activité ou tâche | Description | Terminé ? | Informations supplémentaires |
|----|-----------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Valider que votre environnement inclut toutes les conditions préalables les équipes | Équipes dépend des autres plates-formes pour construire une solution de collaboration de bout en bout. Travailler avec vos équipes informatiques pour vous assurer que vous avez déployé et configuré correctement Exchange, SharePoint Online et OneDrive pour les entreprises. | | [Interaction de SharePoint Online et OneDrive Entreprise avec Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/sharepoint-onedrive-interact) <br/><br/>[Interaction entre Exchange et Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/exchange-teams-interact) |
| 2  | Valider que les équipes est activée pour les clients | Équipes est activée par défaut pour toutes les organisations. Accédez à la page **Services et compléments** dans le centre d’administration Office 365 pour vérifier que les équipes est activée pour vos clients et l’activer si nécessaire. | | [Configurer Microsoft Teams dans votre organisation Office 365](https://docs.microsoft.com/MicrosoftTeams/office-365-set-up) |
| 3  | Configurer les rôles et autorisations | Les équipes prennent en charge deux types de rôles : membre et propriétaire. <br/><br/>Après l’ajout d’un membre à une équipe, un propriétaire pouvez également promouvoir un membre pour le rôle de propriétaire. Pour obtenir les meilleurs résultats, nous vous recommandons d’avoir au moins deux propriétaires affectées à chaque équipe. <br/><br/>Par défaut, tous les membres de l’organisation qui possède une boîte aux lettres hébergée sur Exchange Online peuvent créer une équipe. Un utilisateur qui crée une nouvelle équipe reçoit automatiquement le rôle de propriétaire pour cette équipe. <br/><br/>Si vous le souhaitez, vous pouvez configurer des paramètres de groupe d’Office 365 permettant uniquement spécifiques aux utilisateurs de créer de nouvelles équipes. | | [Assigner des rôles et des autorisations dans Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/assign-roles-permissions) <br/><br/>[Groupes Office 365 et Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/office-365-groups) <br/><br/>[Gérer qui peut créer des groupes d’Office 365](https://support.office.com/article/Manage-who-can-create-Office-365-Groups-4c46c8cb-17d0-44b5-9776-005fced8e618) |
| 4  | Configurer les paramètres des équipes à l’échelle du locataire | Vous pouvez configurer certains paramètres d’équipes au niveau du client. Les utilisateurs qui sont activés pour les équipes héritent de ces paramètres à partir de la configuration de clients :<ul><li>Général</li><li>Intégration de la messagerie</li><li>Applications</li><li>Stockage cloud personnalisé</li><li>Appels et réunions</li><li>Messagerie</li></ul>| | [Activer les fonctionnalités Microsoft Teams dans votre organisation Office 365](https://docs.microsoft.com/MicrosoftTeams/enable-features-office-365) |
| 5  | FACULTATIF : Configurer l’accès invité | Vous utilisez l’accès invité dans des équipes de collaborer avec des personnes en dehors de votre organisation en leur accordant l’accès aux équipes et aux canaux. L’accès invité est un paramètre au niveau de clients dans les équipes. Elle est désactivée par défaut. <br/>Activer l’accès invité et configurer les paramètres de l’invité de clients à l’échelle, si votre organisation envisage d’utiliser cette fonctionnalité. | | [Accès invité dans Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/guest-access) |
| 6  | FACULTATIF : Configurer la stratégie d’appellation des équipes | Les équipes s’appuie sur les stratégies d’attribution de noms pour Office 365 groupes lorsque les utilisateurs créent ou modifier les noms d’équipe. <br/><br/>Par défaut, aucune restriction d’affectation de noms n’est appliquées lorsqu’un utilisateur crée une équipe. <br/><br/>Si vous avez besoin d’appliquer des règles pour les noms des équipes, configurer des groupes de 365 Office les stratégies qui s’appliquent à votre organisation d’appellation. Vous pouvez définir obligatoires préfixes et suffixes et spécifier des mots interdits. | | [Planifier les groupes Office 365 lors de la création d'équipes dans Microsoft Teams](https://docs.microsoft.com/microsoftteams/plan-office-365-groups) <br/><br/>[Stratégie d’attribution de noms de groupes de Office 365](https://support.office.com/article/Office-365-Groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552) |
| 7  | Configurer Exchange pour le domaine SMTP des équipes | Équipes utilise Exchange en ligne pour envoyer des notifications aux membres de l’équipe en utilisant le domaine SMTP — email.teams.microsoft.com — lorsqu’ils avez ajoutés ou supprimés. <br/><br/>Veillez à ajouter un domaine SMTP dans la liste des domaines acceptés dans votre infrastructure Exchange. | | [Ajouter le domaine SMTP de Microsoft Teams comme domaine accepté dans Exchange Online](https://docs.microsoft.com/MicrosoftTeams/smtp-accepted-domain) |
| 8  | Configurer et gérer l’accès des utilisateurs à des équipes | Bien que nous recommandons vivement d’activer tous les utilisateurs pour les équipes, vous pouvez autoriser ou interdire l’accès aux équipes sur une base par utilisateur par affectation ou suppression de la licence de produit des équipes. | | [Gérer l'accès des utilisateurs à Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/user-access) |
| 9  | Attribution de licences aux utilisateurs | Attribuer des licences à vos utilisateurs d’audioconférence, système téléphonique et les Plans d’appel | | [Attribuer des licences Skype Entreprise et Microsoft Teams](https://docs.microsoft.com/SkypeForBusiness/skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses) <br/><br/>[MyAdvisor – scripts d’activation utilisateur](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=5_2_0_6,5_2_0_3) |
| 10 | Facultatif : Utiliser PowerShell administrer des équipes | Vous pouvez utiliser les applets de commande PowerShell, plutôt que le centre d’administration Office 365 pour administrer et gérer les paramètres des équipes. | | [Les équipes Microsoft PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) |
