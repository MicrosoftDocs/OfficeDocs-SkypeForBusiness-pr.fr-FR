# <a name="list-of-session-border-controllers-certified-for-direct-routing"></a>Liste des contrôleurs de frontière de session certifiés pour le routage direct

Microsoft établit des partenariats avec certains fournisseurs de contrôleurs SBC afin de certifier leurs SBC pour une utilisation avec un routage direct. 

Microsoft collabore avec chaque fournisseur pour :

- Travaillez conjointement sur les protocoles d'interconnexion SIP.
- Effectuez des tests intenses en utilisant un laboratoire tiers. Seuls les périphériques qui ont réussi les tests sont certifiés. 
- Exécutez des tests quotidiens avec tous les périphériques certifiés dans les environnements de production et de pré-production. La validation des périphériques en préproduction permet de s'assurer que les nouvelles versions de code de routage direct dans le cloud fonctionneront avec les SBC certifiés. 
- Établissez un processus de support conjointe avec les fournisseurs SBC.


  > [!NOTE]
  > Microsoft prend uniquement en charge phone system si un ou plusieurs périphériques certifiés sont connectés via un routage direct. Microsoft se réserve le droit de refuser les cas de prise en charge lorsqu'un appareil non certifié est connecté au système téléphonique via un routage direct. Si Microsoft détermine que le problème de routage direct d'un client est avec l'appareil SBC d'un fournisseur, le client devra faire appel au fournisseur SBC pour obtenir de l'aide.

Le tableau suivant reprend les périphériques certifiés pour le routage direct. (Pour plus d'informations sur les fournisseurs SBC qui supportent l'optimisation des médias locaux, voir Configurer l'optimisation des médias locaux pour [le routage direct.)](direct-routing-media-optimization-configure.md)

[En savoir plus sur le routage direct](https://aka.ms/dr). Si vous avez des questions concernant le programme de certification SBC pour le routage direct, contactez le drsbccertification@microsoft.com.
