# <a name="manage-a-microsoft-teams-rooms-console-settings-remotely-with-an-xml-configuration-file"></a>Gérer à distance les paramètres d’une console Salles Microsoft Teams avec un fichier de configuration XML

Cet article décrit la gestion à distance des paramètres par défaut utilisés par un appareil Salles Microsoft Teams, y compris l’application d’un thème personnalisé. Il explique comment créer un fichier de paramètres maître et des liens vers des discussions sur la façon de les placer en fonction des besoins sur salles Teams.
  
Il est possible de modifier les paramètres par défaut de salles Teams en mettant à jour un fichier XML maître et en envoyant des copies aux appareils salles Teams distants.

> [!NOTE]
> Certaines fonctionnalités sont disponibles uniquement sur les appareils salles Teams auxquels une licence Salles Microsoft Teams Pro a été attribuée. Pour savoir quelles fonctionnalités nécessitent Salles Microsoft Teams Pro, consultez [licences Salles Microsoft Teams](rooms-licensing.md).
  
## <a name="create-an-xml-configuration-file"></a>Créer un fichier de configuration XML

N’importe quel éditeur de texte peut être utilisé pour créer un fichier de paramètres. Le tableau **Éléments XML** décrit les éléments indiqués dans cet exemple SkypeSettings.xml fichier de configuration (nom de fichier obligatoire).
  
```XML
<SkypeSettings>
  <AutoScreenShare>1</AutoScreenShare>
  <HideMeetingName>1</HideMeetingName>
  <AutoExitMeetingEnabled>true</AutoExitMeetingEnabled>
  <AudioRenderDefaultDeviceVolume>70</AudioRenderDefaultDeviceVolume>
  <AudioRenderCommunicationDeviceVolume>30</AudioRenderCommunicationDeviceVolume>
  <UserAccount>
    <SkypeSignInAddress>username@microsoft.com</SkypeSignInAddress>
    <ExchangeAddress>username@microsoft.com</ExchangeAddress>
    <DomainUsername>domain\username</DomainUsername>
    <Password>Password!</Password>
    <ConfigureDomain>domain1, domain2</ConfigureDomain>
    <ModernAuthEnabled>true</ModernAuthEnabled>
  </UserAccount>
  <TeamsMeetingsEnabled>true</TeamsMeetingsEnabled>
  <SfbMeetingEnabled>true</SfbMeetingEnabled>
  <IsTeamsDefaultClient>true</IsTeamsDefaultClient>
  <WebExMeetingsEnabled>true</WebExMeetingsEnabled>
  <ZoomMeetingsEnabled>true</ZoomMeetingsEnabled>
  <UseCustomInfoForThirdPartyMeetings>true</UseCustomInfoForThirdPartyMeetings>
  <CustomDisplayNameForThirdPartyMeetings>guestname</CustomDisplayNameForThirdPartyMeetings>
  <CustomDisplayEmailForThirdPartyMeetings>guest@microsoft.com</CustomDisplayEmailForThirdPartyMeetings>
  <BluetoothAdvertisementEnabled>false</BluetoothAdvertisementEnabled>
  <AutoAcceptProximateMeetingInvitations>true</AutoAcceptProximateMeetingInvitations>
  <CortanaWakewordEnabled>true</CortanaWakewordEnabled>
  <DualScreenMode>0</DualScreenMode>
  <DuplicateIngestDefault>true</DuplicateIngestDefault>
  <DisableTeamsAudioSharing>true</DisableTeamsAudioSharing>
  <FrontRowEnabled>true</FrontRowEnabled>
  <FrontRowVideoSize>medium</FrontRowVideoSize>
  <SingleFoRDefaultContentLayout>1</SingleFoRDefaultContentLayout>
  <DefaultFoRExperience>0</DefaultFoRExperience>
  <ShowMeetingChat>true</ShowMeetingChat>
  <EnablePublicPreview>false</EnablePublicPreview>
  <NoiseSuppressionDefault>1</NoiseSuppressionDefault>
  <SendLogs>
    <EmailAddressForLogsAndFeedback>username@microsoft.com</EmailAddressForLogsAndFeedback>
    <SendLogsAndFeedback>True</SendLogsAndFeedback>
  </SendLogs>
  <Devices>
    <MicrophoneForCommunication>Device1</MicrophoneForCommunication>
    <SpeakerForCommunication>DeviceX</SpeakerForCommunication>
    <DefaultSpeaker>DeviceX</DefaultSpeaker>
    <ContentCameraId>Camera1</ContentCameraId>
    <ContentCameraEnhancement>true</ContentCameraEnhancement>
    <ContentCameraInverted>false</ContentCameraInverted>
  </Devices>
  <Theming>
       <ThemeName>Custom</ThemeName>
       <CustomThemeImageUrl>file name</CustomThemeImageUrl>
       <CustomThemeColor>
            <RedComponent>100</RedComponent>
            <GreenComponent>100</GreenComponent>
            <BlueComponent>100</BlueComponent>
       </CustomThemeColor>
  </Theming>
  <CoordinatedMeetings enabled="true">
    <TrustedAccounts>username1@microsoft.com,username2@contoso.com</TrustedAccounts>
    <Settings>
      <Audio default="true" enabled="true"/>
      <Video default="true" enabled="true"/>
      <Whiteboard default="true" enabled="true"/>
    </Settings>
  </CoordinatedMeetings>
  <EnableResolutionAndScalingSetting>true</EnableResolutionAndScalingSetting> 
  <MainFoRDisplay> 
      <MainFoRDisplayResolution>1920,1080</MainFoRDisplayResolution> 
      <MainFoRDisplayScaling>100</MainFoRDisplayScaling> 
  </MainFoRDisplay> 
  <ExtendedFoRDisplay> 
      <ExtendedFoRDisplayResolution>1920,1080</ExtendedFoRDisplayResolution> 
      <ExtendedFoRDisplayScaling>100</ExtendedFoRDisplayScaling> 
  </ExtendedFoRDisplay>  
  <EnableDeviceEndToEndEncryption>false</EnableDeviceEndToEndEncryption>
  <SplitVideoLayoutsDisabled>false</SplitVideoLayoutsDisabled>
</SkypeSettings>
```

Si une valeur de variable est de type incorrect, que les éléments sont dans le désordre, que les éléments ne sont pas joints ou qu’une autre erreur est détectée, le fichier XML est *mal formé*. Lors du traitement d’un fichier XML mal formé, les paramètres trouvés jusqu’au point où l’erreur se produit sont appliqués, le reste du fichier est ignoré. Tous les éléments inconnus dans le fichier XML sont ignorés. Si un paramètre est omis, il reste inchangé sur l'appareil. Si une valeur de paramètre n’est pas valide, sa valeur antérieure reste inchangée.

## <a name="manage-console-settings-with-an-xml-configuration-file"></a>Gestion des paramètres de la console à l'aide d'un fichier de configuration XML

Au démarrage, si une console Salles Microsoft Teams trouve un fichier XML nommé SkypeSettings.xml situé dans `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState`, elle applique les paramètres de configuration indiqués par le fichier XML, puis supprime le fichier XML.
  
Selon le nombre d’appareils Salles Microsoft Teams votre entreprise et la façon dont vous choisissez de les configurer, il existe plusieurs façons de placer le fichier de configuration XML. Une fois le fichier poussé sur la console, redémarrez-la pour traiter les modifications apportées à la configuration. Le fichier de configuration XML est supprimé une fois qu';il a été traité. Les méthodes de gestion suggérées pour Salles Microsoft Teams appareils sont décrites dans :
  
- [Configuration de stratégie de groupe pour Salles Microsoft Teams](rooms-operations.md#GroupPolicy)
- [Gestion à distance à l’aide de PowerShell](rooms-operations.md#RemotePS) et [configurer un élément de fichier](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772536(v=ws.11))

Vous pouvez utiliser la méthode que vous souhaitez tant que vous pouvez l'utiliser pour transférer des fichiers et déclencher un redémarrage sur la console. Le fichier doit être lisible, accessible en écriture et supprimé par le compte d’utilisateur local de l’appareil. De préférence, il appartient à cet utilisateur et dispose de privilèges complets. Si les autorisations de fichier ne sont pas définies correctement, le logiciel peut ne pas appliquer les paramètres, peut ne pas supprimer le fichier en cas de traitement réussi et peut même se bloquer.

## <a name="locate-the-content-camera-usb-instance-path"></a>Rechercher le chemin d’accès de l’instance USB de la caméra de contenu

Pour localiser le chemin d’accès de l’instance :

1. Accédez aux paramètres Windows sur la console Salles Microsoft Teams.
2. Entrez le mot de passe administrateur.
3. À partir d’une invite de commandes, tapez `devmgmt.msc` pour afficher Gestionnaire de périphériques.
4. Dans **Gestionnaire de périphériques**, recherchez le nœud **Appareils de création d’images** et recherchez la caméra de contenu.
5. Cliquez avec le bouton droit sur l’appareil photo, puis ouvrez **Propriétés**.
6. Sélectionnez l’onglet **Détails** et recherchez la propriété **Chemin d’accès de l’instance de l’appareil** dans la liste déroulante.
7. La valeur affichée est le chemin d’accès de l’instance d’appareil à définir dans le fichier de configuration XML. Lorsque vous spécifiez le chemin d’accès en XML, remplacez l’esperluette (&) par `&amp;`.

## <a name="set-front-row-as-the-default-layout"></a>Définir la ligne d’avant comme disposition par défaut

Si vous ne définissez pas de disposition d’affichage par défaut pour une salle dans votre configuration XML, la disposition par défaut est définie sur Galerie. Pour voir Ligne de front comme disposition par défaut, ajoutez `<DefaultFoRExperience>1</DefaultFoRExperience>` à votre fichier de configuration XML.

Les utilisateurs finaux peuvent passer de la disposition d’affichage par défaut à l’aide du sélecteur de disposition pendant les réunions.

## <a name="turn-off-front-row"></a>Désactiver la ligne avant

La ligne de front est activée par défaut. Désactivez La ligne front si vous ne souhaitez pas autoriser les utilisateurs finaux à utiliser la ligne front dans une certaine salle. Pour ce faire, ajoutez `<FrontRowEnabled>false</FrontRowEnabled>` à votre fichier de configuration XML.

## <a name="set-front-of-room-scale-and-resolution"></a>Définir l’échelle et la résolution de l’avant de la salle

Pour définir la mise à l’échelle et la résolution de vos affichages front of room, ajoutez `<EnableResolutionAndScalingSetting>true</EnableResolutionAndScalingSetting>` à votre fichier de configuration XML avec le `<MainFoRDisplay>` conteneur. Si votre appareil utilise deux écrans, incluez également le `<ExtendedFoRDisplay>` conteneur. 

Si vous utilisez à la fois des `<MainFoRDisplay>` conteneurs et `<ExtendedFoRDisplay>` pour votre salles Teams à l’aide d’un affichage unique, le `<ExtendedFoRDisplay>` conteneur est ignoré. Pour plus d’informations, reportez-vous à l’exemple xml et au tableau d’éléments ci-dessus.
